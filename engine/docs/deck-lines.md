## 黒黄ティーチ（OP16-080）— 「受けて耐えてライフに仕込む」コントロール

- 基本則（high・3-0×2）: リーダー効果=手札のトリガー持ち1枚でアタック誘導（誘導先に10000/12000がいれば実質無効化）。ライフ追加（6シリュウ/8ティーチ）と二段耐久。手札にトリガー持ち1枚温存の価値が高い。
  - https://note.com/pros_02/n/ndae5700bc10c
- ターン帯（medium・後攻基準）: 後1=サーチ（ラフィット/おれの時代だァ/ハチノス）→後2=4c（デボン/バージェス）orサーチ→後3=6シリュウで回復転換→後4=6シリュウor8ティーチ。
  - https://note.com/hiroshi_info/n/ne34279a83baa
- **ライン: シリュウ・スタック**（high・「一番強い動き」）: 6シリュウ登場→手札1捨て→トラッシュの別シリュウをライフ上に表向き（埋めたシリュウのトリガー=draw2）。6ドン帯以降・要トラッシュにシリュウ。
- ライン: デボン仕込み（high）: 6シリュウ/8ティーチで4デボンをライフへ（トリガー=draw1+トラッシュの1c黒ひげ登場。1c黒ひげ14枚）。※op解決層の対象選択依存＝v1では明示ライン化せず（cpuPick ownBigはシリュウ優先=スタック線と整合）。
- **ライン: 10cティーチ連打**（medium・出典は黒単記事からの転用/黒黄入賞リスト4枚採用が傍証）: 10ドン帯は毎ターン10cティーチ（起動=相手リーダー無効+キャラ無効&アタック不可、12000ブロッカー）。ハチノスはドン不要サーチなので同一ターンに両立可（E46でハチノス設置は既定化済み）。
  - https://tier-one-onepiece.jp/blog/black-teach/ / https://www.mercardop.jp/article-detail/43 / https://cardrush.media/onepiece/decks/1594
- 未確認: マリガン基準／ゼハハ(OP16-116)ライン=**日本語ソースで確立された型として確認できず**（E39の仮定は裏付けなしと判明）。

## 青黄ハンコック（OP14-041）— トリガー登場をドローに変換する受けデッキ

- 基本則（high・公式含む）: リーダーのドローは【ターン1回】なし＝トリガー登場毎にドロー。序盤はライフ1-2枚まで受ける。トリガー登場キャラ17枚（グロリオーサ/マーガレット/ラン/ゴルゴン/シャクヤク）。
  - https://onepiece-cardgame.com/feature/deck/deck_094.php / https://yuyu-tei.jp/show/opc/content/26574
- ターン帯（medium・3-0×3）: 2ドン=グロリオーサでライフ上にトリガー登場キャラを仕込む。理想カーブ「3→5→7→9」= 3:グロリオーサ/マーガレット → 5:4cハンコック → 7:6くまor5ボルサリーノ → 9:9cハンコック。
  - https://cardrush.media/onepiece/articles/27 / https://note.com/vast_auklet210/n/n144d2efd55e9
- **ライン: 4cハンコック・カーブ**（medium）: 5ドン帯は5ボルサリーノより4cハンコック優先（カーブの背骨）。
- **ライン: ナミュール→ゾロ（ゾンビ型）**（medium・単一ソース3-0）: ナミュール(draw2→手札2枚をデッキ上下へ)でクエーサー/8モリアをトップに→4ゾロ(手札1捨てデッキ上1枚をライフへ)/5ボルサリーノ(ライフ2以下)で仕込み→被弾トリガーでトラッシュ蘇生。3-7ドン帯で分割実行可。
  - https://note.com/pros_02/n/n0ff0a1c9614e
- **ライン: 芳香脚リーサル**（high）: 相手ライフ0（または1で2枚）の詰めターンに芳香脚=王下七武海に+2000&ブロック不可→リーダーへ。CPU条件: 相手ライフ≤1。
  - https://cardrush.media/onepiece/articles/27 / https://yuyu-tei.jp/show/opc/content/25539
- **使用不可ライン**（公式サンプルとの照合済み）: EB03-026ハンコック+OP13-052マリーゴールド早出し等、公式構築前提のラインは現50枚に不在＝実装しない。

## 黒ヤマト（OP16-079）— トラッシュ蘇生に【速攻】（E48・✅有意採用・ユーザー観察由来）

- ソース: **ユーザーのプレイ観察**（5cおでん過多・モモコンボ不発）＋頻度計測での裏付け（採用前: 9cモモの助 0.03回/側）。
- **ライン: モモの助コンボ**: 5cモモの助(OP16-084)＋しのぶ(OP16-087・登場時自壊→モモをコスト+20)→5cモモ起動（コスト20以上・場のドン9枚以上）→自壊→**トラッシュの9cモモの助(OP16-085)登場**（リーダー効果で速攻・ブロッカー・登場時さらにコスト6以下ワノ国を蘇生）。7ドン支払い・donTotal9以上。
- **ライン: 6cヤマト即起動**: 6cヤマト(OP16-098)登場→即起動（自壊）→**トラッシュの8cヤマト(OP16-096/097)を速攻で登場**。前提=8cヤマトがトラッシュ（既存ハードコードが捨て優先で準備）。
- 実測（lineh vs heuristic・ミラーN=120）: band1 +3.3pt(6/2)／band2 **+6.7pt(9/1 p=0.021★)**＝2帯正方向・合算15/3。**`LINE_PLAY={'_OP16-079':1}`で既定CPUに採用**。
- **E49追補（ユーザー指摘由来・✅採用）**: トラッシュ起点の別経路3本＝①縁切り(OP16-099)で**トラッシュの5cモモ蘇生**→しのぶ→起動→9cモモ ②縁切りで6cヤマト蘇生(速攻)→即起動→8cヤマトも速攻 ③8cヤマト回収型(OP16-097)でしのぶ回収→即登場→盤面の5cモモ起動。実測=E48採用済み比で **+7.5pt(10/1 p=0.012★)/+9.2pt(11/0 p=0.001★)** の2帯有意。**要点: linesは対象steering（`pick:`＝ライン実行中の蘇生/回収対象の優先no・`G._linePick`）とセット**＝無いと既定のパワー最大選択が「おでん」を拾ってコンボ不発になる。
- 注意: 非curatedリーダーは `DECK_PLANS.byLeader` のキーが**番号キー**（`leaderKeyOf`が `'_OP16-079'` を返す）。
